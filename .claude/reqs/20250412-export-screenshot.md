# PRD: 旅行地图纯前端截图导出功能

版本: v1.0.0 | 日期: 2026-04-12 | 负责人: 需求分析师

---

## 1. 概述

为纯前端运行的首尔旅行地图 H5 应用增加截图导出功能，用户可通过 UI 按钮生成并下载高分辨率行程图片。支持三种导出模式：2K 横屏全景图、2K 横屏按天路线图、竖屏长图（含地图+每日详情）。由于项目构建产物为单个独立 HTML 文件且无后端服务，截图方案必须完全基于前端实现。

---

## 2. 研究发现

### 2.1 现有解决方案
- **当前代码库**: 项目已有一个基于 Playwright 的 `screenshot.mjs`，用于构建后生成静态截图。但这是在 Node 环境下运行的 CI 脚本，无法在浏览器端执行。
- **地图库**: 使用 Leaflet + react-leaflet，tile 已设置 `crossOrigin='anonymous'`（在 `SmartTileLayer.tsx` 中），为 canvas 截图扫清了跨域障碍。
- **构建产物**: 使用 `vite-plugin-singlefile` 输出单一 `dist/index.html`，用户直接在浏览器打开，**无法在运行时调用 Node 或 Playwright**。

### 2.2 技术环境
- **考虑的选项**:
  - **html-to-image**: 轻量（~30KB），基于 SVG + foreignObject 渲染 DOM，支持 `pixelRatio` 高清输出，社区使用广泛。
  - **modern-screenshot**: 基于 `html-to-image` 但改进了对复杂 CSS、渐变、shadow 的兼容性，同样支持 `pixelRatio`，API 更现代。
  - **html2canvas**: 老牌方案，但体积大，对 CSS 新特性支持较弱，Leaflet canvas 渲染兼容性一般。
- **推荐方法**: 使用 `modern-screenshot`。它保留了 `html-to-image` 的轻量优势，同时增强了复杂样式的正确渲染能力，适合截 Leaflet 地图这种带大量 CSS 定位/变换的场景。
- **已知限制**:
  - 若第三方 tile 服务器响应头未正确放行 CORS，截图时 tile 可能显示空白。但项目已设置 `crossOrigin='anonymous'`，只需确认运行时 tile 请求不携带额外敏感头即可。
  - `modern-screenshot` 对 `position: fixed` 的截取有边缘 case，导出时建议使用绝对定位容器包裹截图目标区域。

### 2.3 需要进一步研究
- [ ] 验证生产环境 tile CDN 的 CORS 头是否稳定 — 在真机/多端测试导出效果。

## 4. 用户故事

### 4.1 导出全景图

**作为一个** 计划分享旅行攻略的用户，
**我想要** 一键导出 2K 横版全景地图图，
**以便** 我可以在朋友圈/小红书发布高清的完整行程概览。

**验收标准：**
- [ ] Given 用户在地图页面，When 点击"导出图片"按钮并选择"全景图"，Then 系统生成 2560×1440 像素的 PNG，包含完整地图和所有地点标记
- [ ] Given 图片生成中，When 用户等待不超过 5 秒，Then 浏览器自动触发下载，文件名为 `seoul-travel-map.png`
- [ ] 图片顶部或底部叠加标题、日期等旅行元信息

### 4.2 导出按天路线图

**作为一个** 想按天整理游记的用户，
**我想要** 导出当前选中或指定的某一天的 2K 横版路线图，
**以便** 我发布的每日游记配图与当天行程完全一致。

**验收标准：**
- [ ] Given 用户在"线路"视图并选中了某 Day，When 点击"导出图片"并选择"当天路线图"，Then 生成 2560×1440 PNG，地图视角恰好覆盖该日所有 path point
- [ ] 图片上保留该日的地点编号（A/B/C、1/2/3）和交通标签
- [ ] 图片叠加当日标题（如 "Day3 · 江南购物日"）

### 4.3 导出竖屏长图

**作为一个** 习惯在短视频平台发布竖版内容的用户，
**我想要** 导出 1080px 宽度的竖屏长图，
**以便** 我可以把多天行程做成一图流滑动内容。

**验收标准：**
- [ ] Given 用户选择"竖屏长图"，When 系统渲染导出内容，Then 顶部为 1080×1080（或 1080×1440）的地图区，下面按顺序排列每一天的行程卡片（含时间/地点/交通）
- [ ] 总高度根据天数自动撑开
- [ ] 生成的 PNG 宽度为 1080px，高度不限，文件名为 `seoul-travel-itinerary.png`

## 5. 功能需求

### 5.1 UI 入口
- **FR1**: 在 `MapControls` 组件中新增一个"导出"按钮（桌面端放在右上角控制区，移动端放在顶部控制区）。
- **FR2**: 点击导出按钮后，弹出一个浮层/抽屉（`ExportPanel`），提供三种模式选择：
  1. 横屏全景图（2560×1440）
  2. 横屏当天路线图（2560×1440）
  3. 竖屏长图（1080px 宽，高度自适应）
- **FR3**: 导出面板包含"生成并下载"主按钮和精简的加载状态提示。

### 5.2 截图技术实现
- **FR4**: 引入 `modern-screenshot` 作为项目依赖，使用 `toPng` 方法生成图片。
- **FR5**: 截图时创建一个临时的、脱离视口的隐藏容器（`position: fixed; left: -9999px; top: -9999px;` 或 `visibility: hidden`），尺寸严格匹配目标分辨率。
- **FR6**: 对于横屏导出（全景/按天）：
  - 在临时容器内渲染一套简化版的行程内容，包含地图（Leaflet `MapContainer`）、标记、路线、标签。
  - 使用 `pixelRatio: 2` 调用 `toPng`，保证高清输出。
  - 导出完成后立即从 DOM 中移除临时容器并清理 Leaflet 实例（避免内存泄漏）。
- **FR7**: 对于竖屏长图：
  - 临时容器宽度固定 1080px，顶部为地图区（1080×1080 或 1080×1440，根据设计定）。
  - 下面按天数顺序渲染纯信息卡片（无需再次嵌入完整 Leaflet 地图，可复用现有 `SidebarContent` 的行程列表结构或一个简化只读版本）。
  - 卡片内需包含：日期、标题、地点图标+名称、交通方式简述。
- **FR8**: 所有导出图片均需在边缘或顶部叠加关键元信息（城市名、行程日期范围、标题），由 `ItineraryData.metadata` 提供。

### 5.3 下载行为
- **FR9**: 导出成功后，将生成的 data URL / blob 通过临时 `<a>` 标签触发下载，文件命名使用 `seoul-travel-{mode}-{YYYY-MM-DD}.png` 风格，或简化为对应模式的固定文件名。
- **FR10**: 若导出失败（如 tile 加载超时、CORS 问题），在导出面板中显示友好错误提示："地图瓦片加载失败，请检查网络后重试。"

### 5.4 测试要求
- **FR11**: 为 `ExportPanel` 组件及截图触发逻辑编写单元测试：验证面板打开、模式切换、下载触发的行为。
- **FR12**: 手动测试清单：在 Chrome、Safari、Edge 中分别测试三种导出模式，确认图片可正常打开且信息完整。

## 7. 约束与假设

### 约束
- **技术约束**：纯前端运行，无 Node/后端支持；无法使用 Playwright/puppeteer 在运行时截图。
- **地图约束**：Leaflet tile 必须正确返回 CORS 头，否则 `modern-screenshot` 无法将其绘制到 canvas，会导致地图区域空白。
- **体积约束**：`modern-screenshot` 已很精简，但需确认不显著增加最终 HTML 体积。
- **分辨率约束**：横屏输出固定 2560×1440；竖屏宽度固定 1080px，高度自适应。

### 假设
- **假设 1**：用户设备为现代智能手机或 PC，浏览器支持 `Blob`、`URL.createObjectURL` 和 `download` 属性。若不支持， gracefully fallback 为在新标签页打开图片。
- **假设 2**：Leaflet tile 图层在导出时已经全部加载完成。若未加载完成，截图可能缺少部分地图背景，需考虑导出前短暂预渲染等待。
- **假设 3**：用户倾向于使用默认的"显示地点名"、"显示交通"设置导出图片，截图时直接读取当前 UI 的设置状态。
