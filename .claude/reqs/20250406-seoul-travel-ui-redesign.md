## 1. 需求背景与目标

### 1.1 背景
现有首尔旅行地图应用基于 React + TypeScript + Vite + Leaflet 构建，功能已覆盖多日行程展示、地图标记、路线渲染与交通弹窗。但整体视觉风格偏向中性灰与蓝紫渐变，季节感弱、组件造型偏硬朗，与目标用户（计划春季前往首尔旅行的年轻用户）期望的清新、温暖、可爱氛围存在差距。

### 1.2 目标
依据《首尔旅行攻略 - 春天版设计稿》，对应用进行一套完整的**视觉升级 + 桌面端布局重构**，打造：
- **春天感**：低饱和薄荷绿、樱花粉、奶油黄点缀
- **苹果风格**：大圆角、柔和阴影、玻璃拟态（毛玻璃）
- **可爱细节**：酒店小房子、花朵/叶子标记、爱心与表情元素
- **布局重构（桌面端）**：由现有「左侧边栏 + 全屏地图 + Modal弹窗」改为「左侧边栏（行程列表） + 右上方常驻地点详情卡片 + 右下方地图」的双层右侧面板布局
- **响应式一致**：桌面端与移动端保持统一设计语言，移动端因屏幕限制仍使用弹窗展示地点详情

## 2. 现有项目说明

### 2.1 技术栈
- **Framework**：React 18 + TypeScript
- **Build Tool**：Vite 5
- **Map Library**：Leaflet + react-leaflet
- **Styling**：Tailwind CSS (CDN 引入)
- **Testing**：Vitest + React Testing Library
- **Single File Build**：vite-plugin-singlefile

### 2.2 核心组件
| 文件 | 职责 |
|------|------|
| `src/App.tsx` | 根组件，状态管理（城市、activeDay、弹窗、设置） |
| `src/components/MapView.tsx` | Leaflet 地图渲染、自定义 Marker 与 Polyline、路线标签 |
| `src/components/Sidebar.tsx` | 左侧侧边栏：行程列表、酒店信息、路线详情 |
| `src/components/MapControls.tsx` | 地图控制：城市选择、清除路线、查看全景、设置开关 |
| `src/components/TransportModal.tsx` | 交通详情弹窗 |
| `src/components/LocationDetailModal.tsx` | 地点详情弹窗（移动端保留） |
| `src/components/LocationDetailPanel.tsx` | 地点详情常驻卡片（桌面端新增） |

### 2.3 现有布局与样式痛点
**布局层面：**
- 桌面端采用「左侧 Sidebar + 全屏地图」布局，地点详情通过 `LocationDetailModal` 弹窗临时展示
- 设计稿桌面端为「左侧行程列表 + 右上方常驻地点详情卡片 + 右下方地图」，信息层级更清晰，无需频繁打开弹窗
- 移动端现有布局与设计稿相对接近（顶部控制 + 地图 + 底部列表/工具栏），主要差距在视觉风格

**样式层面：**
- 头部与弹窗使用 `from-blue-500 to-violet-500` 渐变，与春天主题不符
- Marker 采用默认定位针 SVG，缺少可爱图标
- 按钮/卡片圆角较小（`rounded-lg`、`rounded-xl`），阴影偏硬
- 页面背景为 `bg-gray-50`，色温偏冷
- 移动端与桌面端顶部控制栏视觉不统一

## 3. 设计稿解读

### 3.1 色彩系统

#### 主色调（春天色系）
| 名称 | 色值 | 用途 |
|------|------|------|
| 薄荷绿 | `#A8E6CF` | 主色、高亮、按钮、选中态 |
| 薄荷绿深 | `#88D8B0` | 悬停、强调 |
| 薄荷绿浅 | `#DCFAED` | 背景、标签、徽章底色 |
| 樱花粉 | `#FFB7B2` | 辅助色、装饰、点缀 |
| 奶油黄 | `#FFDAC1` | 温暖点缀、次要标签 |
| 毛玻璃白 | `rgba(255,255,255,0.7)` | 玻璃拟态卡片背景 |

#### 中性色
| 名称 | 色值 | 用途 |
|------|------|------|
| 页面背景 | `#F5F7FA` | 页面底层背景 |
| 卡片白 | `#FFFFFF` | 卡片、弹窗、侧边栏 |
| 文字主色 | `#2D3436` | 标题、正文 |
| 文字次色 | `#636E72` | 辅助说明 |
| 文字浅色 | `#B2BEC3` | 时间、备注、禁用态 |
| 边框色 | `#DFE6E9` | 分割线、边框 |

### 3.2 字体规范
- 页面标题：`28px / 700`
- 模块标题：`20px / 600`
- 日期标题：`18px / 600`
- 正文：`15px / 400`
- 辅助文字：`13px / 400`
- 小标签：`12px / 500`

> 项目使用系统默认字体，无需引入新字体文件，仅调整字号/字重/颜色。

### 3.3 圆角与阴影规范
- 大卡片/弹窗：`20px`
- 小卡片/日行程卡片：`16px`
- 按钮：`14px`
- 标签/胶囊徽章：`20px`（全圆角胶囊）
- 输入框/选择器：`12px`
- 卡片阴影：`0 4px 20px rgba(0,0,0,0.06)`
- 悬停阴影：`0 8px 30px rgba(0,0,0,0.1)`

### 3.4 玻璃拟态效果
- 顶部导航栏、控制按钮组、侧边栏头部使用 `backdrop-blur` + 半透明白背景
- 背景色：`rgba(255,255,255,0.7)` ~ `0.85`
- 边框：1px 极浅 `rgba(255,255,255,0.5)` 或 `#DFE6E9`

### 3.5 可爱图标元素（设计稿要求）
- **地点标记**：小花朵/小叶子图标替代默认定位针
- **酒店图标**：小房子带爱心
- **交通图标**：小飞机/小火车（弹窗与路线标签）
- **装饰**：角落小花、飘落花瓣（地图区域轻装饰，优先级低）

## 4. 布局重构范围与验收标准（方案 B）

### 4.1 桌面端布局重构（核心变更）

#### LR-001：整体容器改为左右分栏
- **范围**：`src/App.tsx` 根布局
- **变更**：
  - `sm` 以上屏幕改为左右两栏：`grid grid-cols-[320px_1fr]`（左侧固定 320px）
  - 左侧保留现有 `Sidebar`（移除 `absolute` 定位，改为正常文档流）
  - 右侧区域再分为上下两层：`grid grid-rows-[auto_1fr]`
    - 上方：`LocationDetailPanel`（常驻地点详情卡片），默认高度自适应，最大高度限制为视口 45%
    - 下方：`MapView`（地图），占据剩余空间
- **验收标准**：桌面端不再有全屏地图，地图始终位于右侧下半部分；左侧边栏不再浮层叠加。

#### LR-002：常驻地点详情卡片（LocationDetailPanel）
- **范围**：新建 `src/components/LocationDetailPanel.tsx`
- **变更**：
  - 将现有 `LocationDetailModal` 的内容重构为**非弹窗卡片组件**
  - 圆角 `rounded-[20px]`，背景卡片白 `#FFFFFF`，带柔和阴影
  - 顶部装饰条/标题区改用白底/毛玻璃底，去除蓝紫渐变
  - 当未选中任何地点时，显示默认占位状态（如提示文字或可爱插画占位）
  - 若地点为 `group`/`hotel_group`，继续展示其子地点列表
- **交互**：
  - 点击地图 Marker 或 Sidebar 中的地点/商圈/酒店项时，更新 `App.tsx` 中的 `selectedLocation` 状态，右侧卡片实时响应
  - 选中日期（`activeDay`）后，默认展示该日期第一个非酒店地点的信息，确保卡片不为空
- **移动端降级**：`sm` 以下屏幕隐藏 `LocationDetailPanel`，继续使用 `LocationDetailModal` 弹窗
- **验收标准**：桌面端地点详情常驻可见，无需打开弹窗；内容展示与现有弹窗等价。

#### LR-003：地图容器适配新布局
- **范围**：`src/App.tsx`、`src/components/MapView.tsx`
- **变更**：
  - `MapView` 在桌面端不再 `absolute inset-0`，而是填充右侧下半部分容器
  - 顶部控制栏（`MapControls`） reposition 到右侧主区域顶部，覆盖地图与详情卡片的上方边缘
  - `MapView` 中的 `MapContainer` 尺寸改为 `w-full h-full`
- **验收标准**：地图在右侧下半部分正确渲染，标记、路线、标签位置正常；缩放与平移不受容器影响。

#### LR-004：Sidebar 去浮层化
- **范围**：`src/components/Sidebar.tsx`
- **变更**：
  - 移除 `absolute`、`shadow-xl`、`translate-x` 等浮层相关类名（仅针对桌面端）
  - 改为左侧固定列内的正常滚动块
  - 移动端保留现有抽屉式行为（`absolute`、`translate-x`、遮罩层）
- **验收标准**：桌面端 Sidebar 是布局的一部分，不会覆盖地图；移动端行为不变。

## 5. 样式优化范围与验收标准

### 5.1 全局与布局

#### FR-001：页面背景色替换
- **范围**：`index.html` body 背景、`App.tsx` 根容器
- **变更**：`bg-gray-50` → `#F5F7FA`（通过 Tailwind arbitrary 或自定义配置实现）
- **验收标准**：页面底层背景显示为柔和的 `#F5F7FA`，无灰冷感。

#### FR-002：统一圆角与阴影
- **范围**：所有卡片、按钮、弹窗、侧边栏、新详情面板
- **变更**：
  - 主卡片/弹窗圆角提升至 `rounded-[20px]`
  - 小卡片/日行程卡片圆角 `rounded-[16px]`
  - 按钮圆角提升至 `rounded-[14px]`
  - 标签使用 `rounded-full`
  - 统一引入柔和阴影类（建议新增自定义 CSS 类 `shadow-soft`、`shadow-soft-hover`）
- **验收标准**：组件圆角与阴影符合 3.3 规范，视觉一致。

#### FR-004：移动端底部工具栏春天化
- **范围**：`MapControls.tsx` 移动端底部工具栏
- **变更**：

#### FR-005：移动端汉堡菜单按钮

#### FR-008：日行程卡片春天化

  - 日期标题字号适当放大（`18px`）
  - 内部路线详情区域背景使用 `#F5F7FA` 或 `#DCFAED/30`
- **验收标准**：选中卡片明显呈现薄荷绿选中态，其余卡片阴影柔和、圆角大。

#### FR-009：路线标签与交互文字颜色
- **范围**：`Sidebar.tsx` 内部的「查看详情」「交通详情」链接
- **变更**：`text-blue-500` → `#88D8B0`（薄荷绿深）或 `#A8E6CF`
- **验收标准**：所有蓝色链接文字替换为薄荷绿色系。

### 5.4 地图标记与路线（MapView）

- **验收标准**：地图上酒店图标为小房子，商圈图标为花朵/叶子，景点为小圆点数字，整体可爱清新。

## 7. 用户故事与验收标准

### US-001：作为计划春季旅行的用户，我打开应用时希望感受到春天的清新氛围
- **验收标准**：
  1. 页面背景为柔和的 `#F5F7FA`
  2. 顶部控制栏呈现毛玻璃胶囊效果
  3. 无任何突兀的冷色蓝紫渐变

### US-002：作为用户，我查看侧边栏时希望行程卡片视觉舒适且选中态明显
- **验收标准**：
  1. 日行程卡片圆角 ≥ 16px，带柔和阴影
  2. 选中日期卡片呈现薄荷绿背景/边框
  3. 酒店卡片使用小房子图标，可爱易识别

### US-003：作为用户，我在地图上查看路线时希望标记图标可爱且路线标签不突兀
- **验收标准**：
  1. 酒店 Marker 为小房子图标
  2. 商圈 Marker 为花朵/叶子图标
  3. 路线标签为毛玻璃胶囊形状
  4. 地图整体无硬朗默认图标

### US-004：作为桌面端用户，我希望地点详情常驻显示，不用频繁打开弹窗
- **验收标准**：
  1. 桌面端（≥640px）右侧上方始终显示 `LocationDetailPanel`
  2. 点击 Marker 或 Sidebar 中任意地点，右侧面板实时更新内容
  3. 选中日期后，面板默认展示当天第一个非酒店地点
  4. 未选中任何地点时，面板显示可爱占位提示

### US-005：作为移动端用户，我打开交通/地点详情弹窗时希望风格与整体应用一致
- **验收标准**：
  1. 弹窗头部无蓝紫渐变
  2. 关闭/确认按钮为薄荷绿圆润按钮
  3. 弹窗圆角 20px，阴影柔和

## 8. 约束与假设

### 8.1 约束
- **Tailwind CSS 通过 CDN 引入**，无法使用 `tailwind.config.js` 扩展主题，需使用 arbitrary values（如 `bg-[#A8E6CF]`）或自定义 CSS 文件
- **vite-plugin-singlefile** 构建产物需保持内联，新增 CSS 文件需被 Vite 正确打包
- **Leaflet Marker 使用 DivIcon + HTML 字符串渲染**，图标替换需通过 inline SVG 或 Emoji 实现，避免引入额外图标库依赖
- **移动端与桌面端需同时适配**，不能破坏现有响应式布局
- **测试覆盖率需保持 ≥ 80%**，视觉类修改可能影响 snapshot/断言，需同步更新测试

### 8.2 假设
- 用户未要求改变数据结构、交互逻辑或地图功能，本次仅做纯样式优化 + 桌面端布局重构
- 设计稿中的部分高保真细节（如真实插画）可用 Emoji/SVG 简化实现，用户接受轻量替代方案
- 春天的低饱和地图底图无需替换，通过 Marker 与 UI 的春天化即可达成整体氛围

### 8.3 红旗检查清单（requirement-workflow/SKILL.md）
- [x] 改动范围是否超出纯样式？—— 涉及桌面端布局重构，但不改变功能逻辑与数据流
- [x] 是否引入新的外部依赖？—— 不引入，使用 inline SVG/Emoji + 自建 CSS
- [x] 移动端是否被遗漏？—— 已覆盖 iOS/Android 底部栏与顶部菜单，地点详情继续用弹窗
- [x] 测试是否需要更新？—— 是，布局重构与视觉类断言均需同步调整

## 9. 研究摘要

### 9.1 现有代码发现
- 当前组件结构清晰，状态集中在 `App.tsx`，样式分散在各组件 Tailwind 类名中
- 两套硬编码的 `from-blue-500 to-violet-500` 渐变分别位于 `Sidebar.tsx` 与两个 Modal 的 Header
- Marker 图标通过 `createCustomMarker` 统一生成，替换图标只需修改该函数内的 HTML 字符串
- 项目无全局 CSS 文件，所有动画/复杂阴影/自定义变量均缺失
- `LocationDetailModal` 内容结构可直接复用，提取为 `LocationDetailPanel` 的渲染核心

### 9.2 设计稿核心洞察
- 设计稿强调「苹果风格 + 可爱元素」，因此在保证圆角/阴影/毛玻璃的同时，加入 Emoji（如 🏠、🌸、🍃）是最低成本实现「可爱」的路径
- 桌面端三栏布局在设计稿中演变为「左侧行程 + 右上方地点详情 + 右下方地图」，**本次重构明确新增右上方常驻 `LocationDetailPanel`**，替代原有的 Modal 弹窗
